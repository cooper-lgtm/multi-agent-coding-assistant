# Prompt Strategy

All prompts in this repository are written in English.

Prompt design principles used here:
- single clear role per prompt
- explicit responsibilities and non-responsibilities
- explicit sections for identity, responsibilities, process, input expectations, output requirements, constraints, and quality criteria
- schema-oriented output requirements
- failure criteria where helpful
- no ambiguous ownership for implementation tasks
- compatibility with fallback model routing

Prompt groups:
- planning controller prompts
- debate role prompts
- implementation worker prompts
- quality gate prompts
